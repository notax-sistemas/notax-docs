# Introdução

Bem-vindo à documentação das APIs da Notax. Aqui você encontra tudo que precisa para integrar nossas soluções na sua aplicação.

## APIs disponíveis

| API                     | Versão | Descrição                                |
| ----------------------- | ------ | ---------------------------------------- |
| [Notax Pay](/notax-pay) | v1     | Pagamentos, contas bancárias e convênios |

## Autenticação

Todas as APIs utilizam autenticação via **Bearer Token**. Inclua o token no header de todas as requisições:

```http
Authorization: Bearer {seu-token}
```

## Base URL

| Ambiente | URL                        |
| -------- | -------------------------- |
| Produção | `https://api.notax.com.br` |

## Formato das respostas

Todas as respostas são retornadas em formato JSON. Erros seguem o padrão
[Problem Details (RFC 9110)](https://datatracker.ietf.org/doc/html/rfc9110), com a lista de
ocorrências no campo `errors`:

```json
{
  "type": "https://tools.ietf.org/html/rfc9110#section-15.5.21",
  "title": "Validation Error",
  "status": 422,
  "errors": [
    {
      "code": "AccountHolder.DocumentInvalid",
      "message": "O documento informado é inválido."
    }
  ]
}
```

Consulte [Ambientes e formato de respostas](/guias/ambientes) para a lista completa de
códigos de status.
