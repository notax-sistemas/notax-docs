API de pagamentos da Notax para emissão de boletos, gestão de titulares, contas bancárias, convênios e webhooks.

Todas as requisições (exceto o cadastro e a autenticação de cliente) exigem autenticação via **Bearer token (JWT)**, obtido em `POST /api/v1/client/auth`. As respostas são em JSON e os erros seguem o padrão Problem Details (RFC 9110).
