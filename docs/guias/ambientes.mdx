# Ambientes e formato de respostas

## URLs base

| Ambiente | URL                        |
| -------- | -------------------------- |
| Produção | `https://api.notax.com.br` |

Todos os caminhos de endpoint são relativos a essa URL base. Por exemplo, o endpoint de
autenticação fica em `https://api.notax.com.br/api/v1/client/auth`.

## Formato das respostas

As respostas são sempre retornadas em **JSON**. Requisições bem-sucedidas usam os códigos
`2xx` apropriados (`200 OK`, `201 Created`, etc.).

## Formato dos erros

Erros seguem o padrão [RFC 7807 — Problem Details](https://datatracker.ietf.org/doc/html/rfc7807):

```json
{
  "type": "https://tools.ietf.org/html/rfc7231#section-6.5.1",
  "title": "Bad Request",
  "status": 400,
  "detail": "Descrição do erro"
}
```

| Campo    | Descrição                                       |
| -------- | ----------------------------------------------- |
| `type`   | URI que identifica o tipo do problema.          |
| `title`  | Resumo legível do erro.                         |
| `status` | Código HTTP correspondente.                     |
| `detail` | Explicação específica desta ocorrência do erro. |

Consulte a lista de códigos de status de cada endpoint na
[referência da API Notax Pay](/notax-pay).
